# Scryfall API Portability Assistant

## About
- This project is inteded to help users more easily utilize Scryfall's Rest-like API to access _Magic: The Gathering_ card information
- Current Scryfall API is very powerful, but requires some technical knowledge in requesting and obtaining the response
  - There are also some limitations on the exported data (exporting as CSV loses some of the columns)
- Audience is _MTG_ players, card game data enthusiasts, potentially collectors/investors due to Scryfall having price data

## Methodology
- user inputs their search query (plain language or via scryfall query language syntax)
- calls the Scryfall API and checks for a valid response
- if valid, returns the requested query in scryfall json format
- json is modified according to internal information structure design (mainly separating fields into defined groups)
- response is previewed to user (including card image grid)
- user can decide to export data in a portable format (json or csv)

## Access
- start application (locally hosted)
- activate backend (webhost this all later)
- input your query
- application calls scryfall api with your query, returns response and preview
- can select your desired export

## Structure
- different categories for the json object
- cardInfo: information relating to the card metadata (cardId, manaCost, typeLine, etc)
- marketInfo: pricing information for the card (as of latest api update)
- gameplayInfo: specific gameplay context information
- metadata: data relating to the schema of the modified json
- accessControl: who exported what

## Example
{
  "cardInfo": {

  }
}
